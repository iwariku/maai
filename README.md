# Maai（間合い）

- 紹介ページURL: https://iwariku.github.io/homepage-maai/

- アプリURL: https://maai-theta.vercel.app/

## 概要

メンバーのリアルタイムな状態（ステータス）をひと目で可視化するポートフォリオアプリです。

チーム内で **「休憩中の人に不要な連絡（電話など）をしてしまい、お互いの作業が中断され無駄なコミュニケーションが発生する」** という課題を解決することを最優先に設計しています。

## 背景（開発のきっかけ）

洋菓子店勤務時、店舗間の電話で以下の問題が頻発していました。

- **折り返しの応酬**: 相手が休憩中 → 折り返し → 今度はこちらが休憩中 → 再び折り返し…と何往復も発生
- **休憩中の電話対応**: 急ぎでない用件でも、休憩中に電話が鳴れば対応せざるを得ない

いずれも「相手のステータスが見えない」ことが根本原因です。ステータスが可視化されていれば、発信側で「今は連絡しないでおこう」と判断でき、これらの問題は発生しません。

この実体験から、**リアルタイムなステータス、特に「休憩中」が一目でわかること**の価値が大きいと考え、本アプリを開発しました。

## 想定する活用シーン

> ⚠️ 物流業 / 美容業のケースは、飲食業と同じような構造の問題が起きると考えた **仮説ベースの想定事例** です。

### 飲食業（原体験）

- 複数店舗を抱える飲食店で、店舗間電話が日常的に発生
- 店長や担当者が休憩中だと折り返しの応酬が起きる
- 緊急でない用件でも休憩中に電話対応してしまう

### 物流業

- 運転中のドライバーに業務電話をかけてしまい、**安全面のリスク** + 受信側の負担になる
- ステータスで「運転中」が見えていれば、急ぎでない用件は運転後に連絡する判断が発信側で自然に働く

### 美容室・サロン

- 複数店舗を展開するサロンで、別店舗から店長やスタイリストを呼ぶ電話がかかる
- 相手が施術中・休憩中だと折り返し対応になり、飲食業と同様の往復が起きる
- 接客中の対応強制で、目の前のお客様の体験にも影響しうる

## 現状（MVP1）と今後

このバージョンはプロトタイプ（**MVP1**）として、以下の状態で公開しています。

- データはフロントエンドの状態管理のみ（ページリロードで初期化される）
- 5名のモックスタッフを初期データとして使用
- 「出勤（デフォルト）」「休憩中」「休み」の3状態のみに絞り、**毎日の出勤・退勤の打刻を不要**にする設計（本アプリは勤怠管理ではなく、あくまで「今声を掛けて大丈夫か」を共有するためのツール）

### 今後追加予定の機能

- **休憩タイマー機能**: 「休憩中」へ切り替える際に休憩時間（例: 15分・30分・60分）を設定できる。タイマーが切れたら自動的に「出勤」状態に戻るため、**休憩後の戻し忘れを防ぎ、ステータスの正確性を保てる**
- **サーバー連携 / 認証・認可**: 実データでの運用
